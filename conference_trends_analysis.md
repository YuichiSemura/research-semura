## 3. ICSE トレンド詳細 (2023-2025)

### 3.1 静的解析 + LLM の融合

**ICSE 2025で開催されたSTATICワークショップ**が象徴的です。研究者と産業界が静的解析技術について議論する場として設立されました。

#### 代表的な論文テーマ

**1. LLMによるプログラム修正 (Program Repair)**
- **RepairAgent** (ICSE 2025): 自律的なLLMベースの修正エージェント。従来手法で修正できなかった39件のバグを修正。コストは1バグあたり約14セント
- **D4C** (ICSE 2025): LLMの出力をプログラム修正に最適化。Defects4Jで180件のバグを修正（サンプリング10回のみ）
- **MANIPLE** (ICSE 2025): プロンプトに含める「事実」の最適選択問題を定義

**2. 静的解析によるLLM支援**
- **Planning LLM for Static Detection of Runtime Errors** (ICSE 2025): LLMは静的なコード表現で動作するため、動的な挙動の推論が苦手という課題に対処
- **Monitor-Guided Decoding** (NeurIPS 2023): 静的解析とLLMを組み合わせたコード補完

**3. 脆弱性検出**
- **Vulnerability Detection with Code Language Models** (ICSE 2025): コードLMの脆弱性検出能力を体系的に評価。Claude 3.5, GPT-4o等を含む

### 3.2 年別トピック推移

#### ICSE 2023
- LLM黎明期：ChatGPTの登場直後
- 主なテーマ：
  - 事前学習済みモデルのコードへの応用
  - 自動プログラム修正
  - テスト生成
  - コード類似度検出

#### ICSE 2024
- LLM全盛期：AI/LLMが会議全体の主要テーマに
- 主なテーマ：
  - **UniLog**: LLMによる自動ロギング
  - **Large Language Models for Test-Free Fault Localization**
  - 静的解析ツールの価値の再検証

#### ICSE 2025
- エージェント時代の始まり
- 主なテーマ：
  - **自律エージェント** (RepairAgent等)
  - 静的解析とLLMの統合手法
  - 実用的な脆弱性検出

## 9. 2026年以降のトレンド予測

### 9.1 確実視されているトレンド

#### **Agentic AI（自律エージェント）の本格普及**

2025年が「エージェント元年」なら、**2026年は「マルチエージェントシステムが本番環境へ」の年**と予測されています。

| 予測 | 出典 |
|------|------|
| 2026年末までに企業アプリの**40%**がAIエージェントを組み込む | Gartner |
| 2027年までにSEチームの**55%**がLLMベース機能を構築 | Gartner |
| 2028年までにエンジニアの**90%**がAIコードアシスタント使用 | Gartner |
| Agentic AI市場: 2030年までに**$52B**（現在$7.8B） | 業界予測 |

**具体的な変化:**
- エンジニアの役割が「コードを書く」から「エージェントを調整する」にシフト
- アーキテクチャ設計、システム設計、戦略的判断に人間が集中

### 9.2 ICSE 2026 の動向

**開催情報:** 2026年4月12-18日、リオデジャネイロ（ブラジル）

**投稿状況（既に決定済み）:**
- Cycle 1: 660件投稿 → 60件直接採択 + 101件Major Revision後採択
- Cycle 2: 809件投稿 → 72件直接採択 + 88件Major Revision後採択
- **合計: 1,469件投稿 → 321件採択（採択率約22%）**

**注目ワークショップ: LLM4Code 2026**
- LLMをコード関連タスクに適用する研究の専門ワークショップ
- コード生成、理解、テスト生成、プログラム修正などを対象

### 9.3 静的解析・形式手法の新方向

#### **Neuro-Symbolic Approaches（ニューロシンボリック手法）**

LLMと従来の静的解析・形式手法を組み合わせる「ハイブリッド」アプローチが主流に。

**代表的研究:**
- **IRIS**: LLM + 静的解析によるリポジトリ全体の脆弱性検出
  - GPT-4使用時、CodeQL（最高性能の静的解析器）より**103.7%多い脆弱性**を検出
- **AutoSpec**: 静的解析 + プログラム検証でLLMの仕様合成を強化
  - 既存手法の**1.59倍**のプログラムを自動検証

#### **Vericoding（形式検証されたコード生成）**

POPL 2026で提案された新概念。「Vibe Coding」（自然言語から潜在的にバグのあるコード生成）に対し、**形式仕様から検証済みコード**を生成。

- Dafny検証: LLMの進歩により1年で68% → **96%**に精度向上
- 形式手法研究者にとって大きなチャンス

### 9.4 予測される研究テーマ（2026-2027）

| テーマ | 予測熱度 | 説明 |
|--------|----------|------|
| **マルチエージェント協調** | ★★★★★ | 複数のAIエージェントが協調してソフトウェア開発 |
| **エージェントのガバナンス・セキュリティ** | ★★★★★ | CISOの最大の懸念。セキュリティ確保が競争優位に |
| **LLM + 形式検証** | ★★★★☆ | Vericoding、仕様合成の自動化 |
| **エージェントコスト最適化** | ★★★★☆ | 高コストモデル＋低コストモデルの階層構造 |
| **コード品質・技術的負債** | ★★★☆☆ | AI生成コードの保守性、可読性の評価 |

### 9.5 警告: 失敗リスク

Gartnerは**「2027年までにAgentic AIプロジェクトの40%が失敗する」**と予測。

**失敗の主因:**
- 壊れたプロセスを自動化してしまう（技術ではなく運用の問題）
- ガバナンス不足でセキュリティインシデント発生
- 期待値と現実のギャップ

**研究への示唆:**
- 「AIで自動化すれば良い」だけでなく、プロセス設計の研究も重要
- エージェントの監視・制御手法が研究テーマとして有望

### 9.6 静的解析研究者への具体的提言

#### 今から始められる研究方向

1. **LLMの出力を検証する静的解析**
   - AI生成コードの安全性・正確性検証
   - LLMの「幻覚」を検出する手法

2. **静的解析結果をLLMに効果的に伝える手法**
   - 「Lost in the Middle」問題への対処
   - コンテキスト圧縮・要約技術

3. **形式仕様の自動生成**
   - LLMによる仕様合成 + 静的解析による検証
   - AutoSpecのような反復的改善手法

4. **エージェントの行動検証**
   - AIエージェントが生成・実行するコードの監視
   - 意図しない動作の検出
